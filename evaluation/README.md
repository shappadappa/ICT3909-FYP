# Evaluation

This module provides the evaluation framework for comparing the different meal planning engines. It includes:
- Metric functions for evaluating the quality of meal plans generated by the engines (`metrics.py` and its corresponding unit tests in `test_metrics.ipynb`).
- A comprehensive evaluation notebook (`evaluation.ipynb`) that, when run, evaluates all planners in the various defined scenarios and exports results to the `results/` folder.
- Visualisation of the results of the evaluation (within `evaluation_results.ipynb`), which not only visualises the results of the evaluation in the notebook, but also exports these figures to the `figures/` folder.
- A helper notebook (`evaluation_latex_table_writer.ipynb`) that generates LaTeX code for tables comparing the performance of the different engines across the various scenarios, which is then used in the final report. These tables are exported to the `latex/` folder.

## Metrics

The following metric functions are declared in `metrics.py`, each returning a score in the range [0, 1]:

| Metric | Description |
|--------|-------------|
| `get_ingredient_utilisation_score()` | The proportion of total pantry stock (by weight) consumed by the meal plan - higher is better. |
| `get_expiry_weighted_utilisation_score()` | Like ingredient utilisation, but weights each ingredient by urgency (`1 / days_to_expiry`), prioritising soon-to-expire items - higher is better. |
| `get_food_waste_score()` | The proportion of pantry stock that expires within the planning week but is not consumed - lower is better. |
| `get_dietary_constraint_compliance()` | A binary check (0 or 1) for whether all recipes in the meal plan comply with the user's dietary constraints. |
| `get_nutritional_target_score()` | Measures how closely daily calorie and protein totals match the user's targets, based on mean relative error across the week - higher is better. |
| `get_budget_efficiency()` | The ratio of weekly budget to the cost of ingredients not already available in the pantry, capped at 1.0 - higher is better. |
| `get_pantry_coverage_score()` | The proportion of total required ingredient quantities that are already available in the pantry - higher is better. |
| `get_variety_score()` | The number of unique recipes divided by total meal slots - higher is better. |

## Results

All evaluation outputs are saved to the `results/` folder:

| File | Description |
|------|-------------|
| `planner_metrics.json` | Per-scenario metric values for each planner |
| `average_planner_metrics.json` | Metric values averaged across all scenarios for each planner |
| `best_fitness_scores.json` | Best fitness scores achieved per scenario and planner |
| `average_fitness_scores.json` | Average fitness scores across scenarios for each planner |
| `best_meal_plans.json` | The best meal plan (names only) generated per scenario and planner |
| `ga_convergence_values.json` | Fitness convergence data from the GA planner across generations |
| `ilp_solve_statuses.json` | Solver status codes returned by the ILP planner for each scenario |
